Versión 5.0.1
-------------
- Corregido: La lista de adaptadores de red en Windows Vista/7 incluía tambien los adaptadores
  desactivados.
- Corregido: El programa no detectaba a veces la lista de adaptadores de red, especialmente en
  Vista/7.
- Corregido: En ciertas situaciones el programa fallaba al guardar la configuración para todos
  los usuarios.
- Corregido: Al usar la misma configuración para todos los usuarios el programa mostraba una
  confirmación del Control de Cuentas de Usuario antes de activar la planificación de apagado,
  incluso si la configuración no había cambiado.
- Corregido: El programa no guardaba correctamente el valor AM/PM de la hora de cierre.

Versión 5.0.0
-------------
- Nuevo modo de cierre: Inactividad de usuario (sin actividad desde el teclado o raton).
- Nuevo modo de cierre: Inactividad de red (transferencia de entrada y/o salida).
- Nueva opción de planificación: Ahora se puede seleccionar mas de una condición de apagado a la
  vez (por ejemplo, apagar cuando no haya actividad de CPU ni de red).
- Mejorada la compatibilidad con Windows Vista/7.
- Opción de promediar los valores de uso de CPU para minimizar el efecto de valores de pico.
- Cambios en la ventana de aviso de apagado:
  · Añadido un botón de Reconfigurar.
  · El botón de Reactivar es reemplazado con el botón de Postponer para las condiciones de apagado
    "Fecha/hora" e "Intervalo".
  · Al usar cualquiera de los botones de esa ventana se solicitará la contraseña administrativa
    (si se ha establecido alguna).
- Cambios en la ventana de display:
  · Varios ajustes para reflejar mejor el cierre planificado.
  · Opción de establecer transparencia.
  · Modificados (de nuevo) los colores de fondo de los gráficos para hacerlos mas legibles.
- Corregido: Auto-reactivar cierre no funcionaba con Bloquear sesión.
- Corregido: el programa comprobaba la fecha de cierre incluso cuando no se usaba la condición
  de "Fecha/hora"
- Corregido: Al usar la condición de "Fecha/hora", el cierre no se realizaba si se cambiaba el
  reloj del sistema a una hora posterior a la hora de cierre.
- Corregido: Si el salvapantallas estaba activado a la hora del cierre, la captura de pantalla
  incluía el salvapantallas en lugar de los contenidos del escritorio.
- Algunos otros pequeños cambios y correcciones.

Version 4.50
------------
- Añadios cierre y reinicio administrativos, tanto en en equipo local como en equipos en red.
- Mejorada la proteccion del programa: Si el programa es abortado, puede reiniciarse a si mismo
  automaticamente o puede iniciar un cierre de emergencia (ahora tambien para Windows 2000/XP/Vista).
- Interfaz mejorado: Añadida una barra de menus y movidos los botones de accion a una barra de
  herramientas. Además, ahora se puede activar y desactivar el apagado desde el icono de la barra de
  tareas.
- Añadido un boton en la ventana de confirmacion de apagado para reactivar/postponer el cierre.
- Si el programa está activo y es ejecutado de nuevo, la copia actual es activada en vez de mostrar
  un mensaje de aviso.
- Modificado el color de los graficos del display para hacerlos mas legibles.
- Arreglado algunos problemas que impedian usar la tecla Windows como atajo de teclado.
- Añadido el archivo de ayuda.
- Algunos otros pequeños cambios y correcciones.

Version 4.20
------------
- Soporte completo para Temas de Escritorio de Windows XP (usando Windows XP Theme Manager
  by Mike Lischke).
- Añadida una opción para reactivar o no explícitamente el ultimo cierre programado.
  Esto permite lanzar el programa al inicio de sesión sin reactivar el ultimo cierre.
- Relacionado con el punto anterior, la reactivación del cierre funciona ahora con todos
  los modos.
- Cambiada la forma en el que el cierre por inactividad de CPU funciona: ahora está basada
  en el porcentaje de uso de la CPU en vez de usar el porcentaje de inactividad (que era
  bastante confuso para algunas personas).
- El menú/botón "Cerrar ahora" muestra ahora un menú para elegir el tipo de cierre en vez
  de usar el ultimo tipo de cierre usado.
- La fecha de cierre (si se usa) es mostrada ahora en el display.
- Arreglado: programar un cierre no impide el cierre del sistema desde el Menú Inicio.
- Arreglado: ahora se pueden se pueden especificar parámetros de linea de comandos para el
  programa a ejecutar antes del cierre.
- Algunos otros pequeños cambios y arreglos.

Version 4.10
------------

  botones de Aceptar o Buardar. Ahora, esos cambios son temporales.
